## 项目自动执行规则

以下规则在每次对话中自动生效，无需手动 @ 引用或斜杠命令触发。

### 规则 1：命令行错误 — 自动防范 + 自动记录
**触发时机**：每次执行命令行操作时
- **执行命令前**：如果项目中存在 `logs/命令行错误记录.md`，先快速浏览其中的历史错误，在本次操作中主动避免重犯同类错误
- **命令出错后**：自动将错误追加到 `logs/命令行错误记录.md` 的对应分类下，格式如下：
  ```
  #### 错误 #[编号] — [简短描述]
  - **日期**：YYYY-MM-DD HH:MM
  - **错误命令**：`[实际执行的错误命令]`
  - **错误现象**：[报错信息或异常行为]
  - **根因分析**：[为什么会犯这个错误]
  - **正确做法**：`[应该使用的正确命令]`
  - **教训**：[一句话总结]
  ```

### 规则 2：对话记录 — 完成重要任务后自动追加
**触发时机**：完成用户的一个重要任务（涉及关键决策、技术方案、架构设计、重要代码变更、Bug 修复、待办计划）后
- 如果项目中存在 `logs/对话记录.md`，自动将本次任务的重要内容追加到该文件末尾
- 按日期分组（`## YYYY-MM-DD`），每条记录格式：
  ```
  ### [简短主题标题]
  - **时间**：HH:MM
  - **类别**：[决策 | 问题解决 | 技术方案 | 代码变更 | 待办事项 | 知识点 | 其他]
  - **摘要**：[2-3 句话概括]
  - **详情**：[要点列表]
  - **相关文件**：[涉及的文件路径]
  ```
- **筛选标准**：只记录重要内容（决策、方案、变更、Bug 修复），不记录简单问答和调试细节
- 完成记录后，在回复末尾用一行简要告知：`> 📝 已自动记录：[条目标题]`

### 规则 3：代码文档 — 创建/修改文件后自动更新
**触发时机**：创建新的代码文件 或 对现有代码文件做了重大修改后
- 检查该文件所在目录的 `README.md` 是否已记录此文件
- 如果未记录，自动将文件说明追加到对应 `README.md` 的文件列表和详细说明中
- 同时更新 `docs/代码文档管理总控.md` 底部的文件状态追踪表
- 完成后在回复末尾用一行简要告知：`> 📋 已自动更新文档：[README 路径]`

### 规则优先级与注意事项
- 规则 1（错误防范）优先级最高，每次执行命令前必须检查
- 规则 2 和 3 在完成用户请求后静默执行，不影响主要回复的节奏
- 如果某条规则的目标文件不存在（如项目未初始化），则跳过该规则，不报错
- 自动记录/更新时不要修改或删除已有内容，只做追加
