## 依赖关系图

```
A 基础数据 ──────────────────────────────────── 无依赖
C 汇率管理 ──────────────────────────────────── 无依赖

B 报价管理 ──→ A（Region、FeeCategory）

D 数据导入 ──→ A（Customer、Region）
           ──→ B（pricing_parser 写入 PricingVersion/PricingRule）
           ──→ B（RemotePostcode 用于偏远费匹配）

E 订单管理 ──→ A（Customer、Region）
           ──→ B（pricing_engine 读取计价规则）
           ──→ C（ExchangeRate 汇率换算）

F 导出管理 ──→ A（Customer — 区分RMB/EUR结算模式）
           ──→ C（ExchangeRate）
           ──→ E（Order、OrderFee 数据）
```

**修改影响速查**：

| 我要改… | 需要检查的下游模块 |
|---------|------------------|
| A 基础数据 | B、D、E、F（但只要不改模型字段名，影响极小） |
| B 报价管理 | D（报价导入逻辑）、E（费用计算） |
| C 汇率管理 | E（费用计算时查汇率）、F（导出时查汇率） |
| D 数据导入 | 无下游（改 D 不影响其他模块） |
| E 订单管理 | F（导出读取订单数据） |
| F 导出管理 | 无下游（改 F 不影响其他模块） |

## v6 架构改造要点（2026-04-20 ~ 2026-04-21）

### 0. 统一订单入口 — 单运单号单Order

**核心变更**：`waybill_no` 改为唯一约束（去掉与 `bill_period` 的联合唯一），同一运单跨期导入时只更新不新建。

| 变更前 | 变更后 |
|--------|--------|
| 同运单号+不同账期 = 多条 Order | 同运单号 = 唯一 Order，跨期合并 |
| 跨期数据靠查询合并 | 单 Order 包含所有数据 |
| `(waybill_no, bill_period)` 联合唯一 | `waybill_no` 单独唯一 |

**新增字段**：
- `Order.import_periods`（TEXT）：逗号分隔的账期列表，记录该订单出现过的所有账期
- `Order.needs_second_delivery`（BOOLEAN）：二派费标记

**跨期时间线**：虽然 Order 合并为一条记录，但详情页仍展示跨期时间线。通过 `import_period_list` 和 `OrderFee.import_period` 字段匹配，展示每个账期导入了哪些费用。

**COD手续费预收取**：
- 适用国家：PL、IT、AT、DE（`COD_COUNTRIES`）
- 有头尾程运费但无COD确认金额时 → 预收最低COD手续费（`min_amount`）
- 确认后 → `max(3%×abs(cod_amount), min_amount)`
- "签收"定义：订单出现在某期代理的COD表上

### 0.1 新增二次派送费科目（SECOND_DELIVERY）

**科目信息**：code=`SECOND_DELIVERY`，name=`二次派送费`，group=`杂费`

**触发条件**：订单出现在代理账单的「二派费」或「二次派送」Sheet中，标记 `needs_second_delivery=True`

**计算逻辑**：使用代理给出的 `input_amount`（EUR）× 汇率 → RMB

**涉及修改**：
- `seed_data.py` — 新增科目定义
- `models.py` — `needs_second_delivery` 字段 + `applicable_fees` 属性
- `excel_parser.py` — Sheet映射 `second_delivery → SECOND_DELIVERY`（原为MISC_FEE）+ 标记
- `pricing_engine.py` — 新增 `SECOND_DELIVERY` 计算分支
- `orders.py` — 筛选逻辑 + 标签
- `order_detail.html` — 科目状态 + 标记显示
- `orders.html` — 蓝色Badge + 筛选选项
- `__init__.py` — 自动迁移 `needs_second_delivery` 列

| 科目 | 计算逻辑 | 关键改动 |

### D — 数据导入（v5重构）

**职责**：接收上传的 Excel 文件，根据文件类型分三条解析路径。

| 层级 | 文件 | 说明 |
|------|------|------|
| 模型 | `models.py` → `ImportLog` | 导入日志（文件名、类型、状态、错误信息） |
| 路由 | `routes/imports_routes.py` | 上传 + 三种文件类型分支 + 报价确认 |
| 服务 | `services/excel_parser.py` | 代理账单解析（以表为单位、列名映射、订单创建） |

| 模板 | `templates/imports.html` | 导入管理页面（三种文件类型切换） |

**关键流程**：

```
文件上传 → file_type 判断
  ├─ agent_bill    → excel_parser.parse_agent_bill()  → Order + OrderFee
  ├─ cargo_status  → cargo_status_parser.parse()      → 更新 Order 邮编/货态/COD金额
  └─ pricing_file  → pricing_parser.preview → 确认 → pricing_parser.commit
```

**代理账单解析改造**：
- 以 Sheet 为单位独立处理，不同 Excel 的相同 Sheet 类型通用
- 列映射改为**列名字典匹配**（非位置匹配），兼容列顺序差异
- 列名别名表支持多版本表头命名

**Sheet→科目映射规则**（优先级从上到下）：
