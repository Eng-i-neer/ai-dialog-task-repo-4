# 命令行错误记录 Workflow

执行以下两项操作：

## 一、回顾与记录（向后看）
1. 回顾当前对话中所有命令行操作，找出犯过的错误
2. 读取 `logs/命令行错误记录.md`，将新错误按类别追加到对应分类下
3. 每条错误包含：日期、错误命令、错误现象、根因分析、正确做法、教训
4. 如果没有犯错，告知用户"本次操作过程未发现命令行错误"

## 二、自查与预防（向前看）
1. 通读 `logs/命令行错误记录.md` 中所有已记录的错误
2. 将其作为检查清单，在后续命令行操作中主动避免重复犯同类错误
3. 如果即将执行的命令与某条历史错误相似，先停下来检查
