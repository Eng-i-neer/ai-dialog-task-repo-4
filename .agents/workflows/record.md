# 对话记录 Workflow

执行以下步骤：

1. 读取 `logs/对话记录.md` 文件，理解其中嵌入的 PROMPT 指引
2. 回顾当前对话的全部内容，识别值得记录的重要信息
3. 按照文档中定义的记录规则（日期分组、标准格式、筛选标准），将重要内容追加到 `logs/对话记录.md` 末尾
4. 完成后告知用户记录了哪些条目，如无重要内容则告知"本次对话无需记录的重要内容"

### 记录标准
- ✅ 记录：关键决策、技术方案选型、架构设计、重要代码变更、Bug 原因与修复方案、待办/后续计划
- ❌ 不记录：闲聊、简单问答、重复调试细节、未采纳的方案
